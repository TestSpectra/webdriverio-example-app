Querying

Examples of querying for DOM elements in WebDriverIO, for a full reference of commands, go to webdriver.io and read Selecting Elements: Best Practices Guide

$() - Element Selection

To query for elements, use the $() selector. 

const button = await $('#query-btn')
await expect(button).toHaveText('Button')

const btn = await $('.query-btn')
await expect(btn).toHaveText('Button')

const firstBtn = await $('#querying .well>button:first-child')
await expect(firstBtn).toHaveText('Button')

To find elements by data attribute, use the attribute selector. 

const elem = await $('[data-test-id="test-example"]')
await expect(elem).toHaveElementClass('example')
Div with data-test-id

You can get element attributes and CSS properties using WebDriverIO methods. 

const elem = await $('[data-test-id="test-example"]')
const attrValue = await elem.getAttribute('data-test-id')
await expect(attrValue).toBe('test-example')

// or you can get an element's CSS property
const position = await elem.getCSSProperty('position')
await expect(position.value).toBe('static')

You can chain multiple assertions together. See assertions documentation. 

const elem = await $('[data-test-id="test-example"]')
await expect(elem).toHaveAttribute('data-test-id', 'test-example')
await expect(elem).toHaveCSSProperty('position', 'static')

Finding Elements by Text

We can find elements by their content using XPath or text selectors. See WebDriverIO documentation 

// Find element containing text
const listItem = await $('.query-list*=bananas')
await expect(listItem).toHaveElementClass('third')

// Using XPath to find by text
const item = await $('//li[contains(text(), "bananas")]')
await expect(item).toHaveElementClass('third')

const firstItem = await $('.query-list*=apples')
await expect(firstItem).toHaveElementClass('first')

// Find element by partial text match
const list = await $('ul*=oranges')
await expect(list).toHaveElementClass('query-list')

const button = await $('button*=Save Form')
await expect(button).toHaveElementClass('btn')

Scoped Element Selection

We can find elements within a specific DOM element using scoped queries. See WebDriverIO documentation 

const form = await $('.query-form')
const firstInput = await form.$('input:first-child')
await expect(firstInput).toHaveAttribute('placeholder', 'Email')

const lastInput = await form.$('input:last-child')
await expect(lastInput).toHaveAttribute('placeholder', 'Password')

Getting Root Element

You can get the root document element or work with specific elements. See WebDriverIO documentation 

// Get the root html element
const html = await $('html')
await expect(html).toBeDisplayed()

// Work with a specific element
const list = await $('.query-ul')
await expect(list).toHaveElementClass('query-ul')

// Get children of an element
const items = await list.$$('li')
await expect(items).toHaveLength(3)

Best Practices: Selecting elements

Prefer dedicated data-test-id attributes to CSS class names and element IDs. See detailed discussion at Best Practices: Selecting elements 

// Worst - too generic, no context
await $('button').click()

// Bad. Coupled to styling. Highly subject to change.
await $('.btn.btn-large').click()

// Average. Coupled to the `name` attribute which has HTML semantics.
await $('[name=submission]').click()

// Better. But still coupled to styling or JS event listeners.
await $('#main').click()

// Slightly better. Uses an ID but also ensures the element
// has an ARIA role attribute
await $('#main[role=button]').click()

// Much better. But still coupled to text content that may change.
const submitBtn = await $('button*=Submit')
await submitBtn.click()

// Best. Insulated from all changes.
await $('[data-test-id=submit]').click()
