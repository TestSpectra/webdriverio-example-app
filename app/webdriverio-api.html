WebDriverIO API

Examples of uses of the WebDriverIO API, for a full reference, go to webdriver.io

Custom Commands

To add a custom command, use browser.addCommand().

browser.addCommand('console', {
  isElement: true
}, function (method = 'log') {
  console[method]('The element is', this)
  return this
})

browser.getCookies()

To manage cookies, you can use commands like browser.getCookies(), browser.setCookies(), and browser.deleteCookies().

await browser.setCookies({ name: 'fakeCookie', value: '123ABC' })
const cookies = await browser.getCookies(['fakeCookie'])
console.log(cookies)
await browser.deleteCookies(['fakeCookie'])

browser.capabilities

To get information about the browser and platform, use browser.capabilities.

console.log(browser.capabilities.platformName)
console.log(browser.capabilities.browserName)

WebDriverIO Configuration

WebDriverIO configuration is managed in the `wdio.conf.js` file. You can access configuration values through `browser.options`.

console.log(browser.options.baseUrl)
console.log(browser.options.waitforTimeout)

element.isDisplayed()

To see whether a DOM element is visible, use element.isDisplayed().

const hiddenP = await $('.dom-p p.hidden')
const visibleP = await $('.dom-p p.visible')

await expect(hiddenP).not.toBeDisplayed()
await expect(visibleP).toBeDisplayed()

I'm hiding!

I'm visible!

Environment Variables

WebDriverIO does not have a built-in environment variable system. You can use a library like `dotenv` to manage environment variables.

// Using dotenv
require('dotenv').config()

console.log(process.env.HOST)
console.log(process.env.API_SERVER)

browser.log()

To add custom log messages, you can use browser.log().

browser.capabilities.platformName

To get the name of the underlying OS, use browser.capabilities.platformName.

console.log(browser.capabilities.platformName)

browser.capabilities.browserVersion

To get the version of the browser being run, use browser.capabilities.browserVersion.

console.log(browser.capabilities.browserVersion)
